# 上下文分层

## 推荐层次

1. 固定规则：系统、项目、工具、安全边界。
2. 任务目标：本轮要完成什么，不做什么。
3. 当前状态：进度、已改文件、未解决问题。
4. 证据材料：代码片段、日志、测试输出、文档摘录。
5. 候选资源：可选 prompt、skill、reference、外部链接。
6. 输出格式：最终交付物结构和验收标准。

## 加载顺序

先加载固定规则和任务目标，再按需要检索资源。不要在任务开始时加载
全部技能正文。搜索工具应返回：

- 资源 id。
- 标题。
- 摘要。
- 标签。
- 评分。
- 命中字段。
- 大小或 token 估算。

只有当资源真的相关时，再读取正文或章节。

## MCP 资源策略

对当前 `vibe-code-mcp`，推荐：

- `search_*` 返回 metadata。
- `get_*` 按 id 精确读取。
- 大 reference 支持 `section` 和 `max_chars`。
- 对每个资源建立 `use_when` 和 `avoid_when`。
- 默认禁用高风险、私有、实验性 prompt。
