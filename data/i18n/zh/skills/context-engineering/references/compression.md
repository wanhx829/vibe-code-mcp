# 上下文压缩

## 压缩什么

保留：

- 用户目标。
- 已确认约束。
- 关键决策及理由。
- 已执行命令和结果摘要。
- 失败原因。
- 未完成事项。
- 相关文件路径和行号。

丢弃或降级：

- 长篇背景解释。
- 重复日志。
- 已关闭的假设。
- 与当前阶段无关的资源全文。
- 可以通过 id 再取回的 reference 正文。

## 压缩模板

```markdown
## Goal

## Constraints

## Decisions

## Evidence

## Files

## Open Questions

## Next Actions
```

## 长任务维护

- 每完成一个阶段，写一次阶段摘要。
- 每次测试失败，记录失败命令、症状、假设和下一步。
- 每次修改计划，记录“为什么改”。
- 恢复任务时先读摘要，再按文件路径补证据。

## 判断压缩质量

压缩后的上下文应该让另一个 AI 能回答：

- 现在要做什么？
- 为什么这样做？
- 已经做过什么？
- 哪些事实已验证？
- 下一步最小动作是什么？
