# Prompt 评估

## 基础指标

- 指令遵循：是否按流程和格式输出。
- 完整性：是否覆盖必要字段。
- 稳定性：同类输入是否得到一致结构。
- 鲁棒性：输入缺失、冲突或噪声时能否处理。
- 成本：prompt 长度和输出长度是否可控。

## 测试集

至少准备：

- 3 个典型输入。
- 2 个边界输入。
- 1 个恶意或冲突输入。
- 1 个信息不足输入。

## 迭代方式

1. 跑样例。
2. 标记失败类型。
3. 只改一个 prompt 模块。
4. 重新跑样例。
5. 保留变更原因。

## 验收标准

一个 prompt 能进入 MCP 默认资源前，应满足：

- 有明确 `use_when`。
- 有明确 `avoid_when`。
- 输出格式稳定。
- 不依赖私有背景。
- 不含高风险、攻击性或版权不明内容。
