# Prompt 模式

## 任务澄清模式

用于需求不完整时：

```markdown
你是需求分析助手。请先识别缺失信息，再给出最小可执行任务说明。

输出：
1. 已知事实
2. 缺失信息
3. 默认假设
4. 风险
5. 可执行任务描述
```

## 计划模式

用于开发前：

```markdown
请将需求拆成实施计划。每个任务必须包含文件、步骤、测试命令、
验收标准和回滚风险。不要写占位符。
```

## 审查模式

用于代码或设计评审：

```markdown
请按严重程度列出问题。每个问题包含位置、影响、原因、修复建议
和需要补充的测试。无问题时明确说明剩余风险。
```

## 压缩模式

用于长上下文：

```markdown
请把以下上下文压缩为任务恢复摘要，只保留目标、约束、决策、
证据、文件、未完成事项和下一步。
```

## 反模式

- 用大量“必须、极其、顶级”代替具体约束。
- 只要求“详细”，不定义输出结构。
- 把多个角色混在一个 prompt 里。
- 同时要求简洁和全面，但不说明优先级。
- 不写失败处理。
