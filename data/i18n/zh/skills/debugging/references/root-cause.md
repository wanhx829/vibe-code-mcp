# 根因定位

## 反向追踪

从出错位置开始问：

- 错误值是什么。
- 谁传入了这个值。
- 上一层为什么生成这个值。
- 哪个边界首次出现异常。
- 哪个变更让这个边界失效。

持续追踪，直到找到源头。

## 多组件系统

对每个组件边界记录：

- 输入。
- 输出。
- 配置。
- 权限。
- 外部依赖状态。

示例边界：

- 前端 -> API。
- API -> 服务。
- 服务 -> 数据库。
- CI -> 构建脚本。
- 构建脚本 -> 签名工具。

## 模式对比

找到同仓库中正常工作的类似代码，对比：

- 调用顺序。
- 参数。
- 配置。
- 错误处理。
- 初始化流程。

不要假设“小差异没影响”。
