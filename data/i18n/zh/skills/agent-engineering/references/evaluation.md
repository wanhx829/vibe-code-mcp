# Agent 评估与安全

## 评估维度

- 任务成功率：最终结果是否满足用户目标。
- 工具调用准确率：工具选择、参数、调用顺序是否正确。
- 恢复能力：工具失败、信息缺失、上下文冲突时能否恢复。
- 成本与延迟：调用次数、token、墙钟时间是否可接受。
- 安全边界：是否越权、泄露敏感信息、执行危险操作。
- 可解释性：是否能回放关键决策和证据链。

## 最小评估集

为每个 Agent 至少准备：

- 3 个 happy path。
- 3 个边界输入。
- 3 个工具失败或外部依赖失败案例。
- 1 个权限或安全风险案例。

## 观测记录

每次运行记录：

- 用户目标。
- 选择的计划。
- 工具调用序列。
- 每次工具输入输出摘要。
- 最终结果。
- 成功/失败标签。
- 失败原因和后续修复动作。

## 安全边界

- 读写工具分离。
- 危险操作要求显式确认。
- 文件、网络、命令执行要有 allowlist。
- 日志中不要保存密钥、完整凭证和个人敏感数据。
- 对外部工具返回内容做不可信输入处理。
