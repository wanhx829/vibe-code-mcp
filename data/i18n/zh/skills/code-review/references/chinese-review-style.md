# 中文 Review 表达

## 分级标注

- `[必须修复]`：不修不能合入。
- `[建议修改]`：应修复，但可结合排期。
- `[仅供参考]`：风格或替代方案。
- `[问题]`：需要作者解释意图。

## 评论模板

```markdown
[必须修复] 标题

这里可能导致什么问题。

原因：

建议：

测试建议：
```

## 表达原则

- 对代码和风险下判断，不评价人。
- 对不确定的问题先提问。
- 对确定的阻塞问题直接标级别。
- 给出可执行建议，而不是只说“不行”。

## 示例

```markdown
[必须修复] 用户输入直接拼接 SQL

这里会带来 SQL 注入风险。建议改为参数化查询，并补充包含特殊字符
输入的测试。
```

```markdown
[问题] 这里使用同步文件读取是出于什么考虑？

如果该路径在请求处理中执行，可能阻塞事件循环。是否可以改为异步
读取，或确认它只在启动阶段运行？
```
