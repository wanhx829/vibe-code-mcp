# RAG Pipeline

## 文档解析

优先保留结构：

- 标题层级。
- 段落。
- 表格。
- 代码块。
- 图片说明。
- 链接和引用。

解析后的每个单元都应带元数据：`source`、`section`、`updated_at`、
`permissions`、`content_type`。

## 分块

分块策略：

- Markdown：按标题层级切分。
- API 文档：按 endpoint 或函数切分。
- 代码：按文件、类、函数切分。
- FAQ：按问题答案对切分。
- 长报告：先章节，再滑窗。

## 检索

常见组合：

- 关键词检索：适合精确术语、错误码、函数名。
- 向量检索：适合语义相似。
- 混合检索：多数生产场景优先。
- Rerank：top-k 较大时提高相关性。

## 生成

生成阶段要明确：

- 只基于检索证据回答。
- 引用来源。
- 证据不足时说明缺少什么。
- 不把推测伪装成事实。
